### **問題の概要**
- `params` の型が `{ projectId: string }` ではなく `Promise<any>` になってしまい、Next.js の型推論により `PageProps` の制約を満たしていない。
- `app` ディレクトリと `pages` ディレクトリが混在しているため、Next.js の型推論が混乱している可能性がある。
- `.next` のキャッシュや TypeScript の設定 (`tsconfig.json`) に問題がある可能性がある。

### **やるべき修正**

#### ✅ **1. `tsconfig.json` の `include` に `app` を追加**
現在の `tsconfig.json` に `app` ディレクトリが含まれていない可能性があるため、以下のように修正してください。

```json
"include": [
  "next-env.d.ts",
  ".next/types/**/*.ts",
  "**/*.ts",
  "**/*.tsx",
  "app"
]
```

---

#### ✅ **2. `pages/` フォルダをリネーム**
`app/` ディレクトリを使用している場合、`pages/` フォルダが影響を与える可能性があるため、一時的にリネームして Next.js の `App Router` の動作を安定させる。

```sh
mv app/pages app/pages_backup
```

その後、型エラーが解消されるか確認する。

---

#### ✅ **3. `generateStaticParams` の型を明示的に修正**
現在の `generateStaticParams` の戻り値の型が Next.js に正しく認識されていない可能性があるため、以下のように明示的に修正してください。

```tsx
export function generateStaticParams(): { projectId: string }[] {
  return [{ projectId: "example-id" }];
}
```

もし型エラーが続く場合は、`Promise.resolve()` を使用して Next.js に `Promise` を期待させる。

```tsx
export function generateStaticParams(): Promise<{ projectId: string }[]> {
  return Promise.resolve([{ projectId: "example-id" }]);
}
```
